# Debugging

Store your bug investigation and resolution notes here.

## Getting Started
Use the debugging template from `templates/debugging-template.md` to document bugs and their fixes.

## Naming Convention
Descriptive names for the issue:
- `auth-token-refresh-issue.md`
- `database-connection-timeout.md`
- `memory-leak-in-api.md`

## Why Document Bugs?
- Remember solutions for similar issues
- Share knowledge with team
- Build a personal troubleshooting guide
- Track time spent debugging
- Learn from patterns

## Tips
- Document as you investigate
- Include error messages and logs
- Note what didn't work
- Explain the root cause clearly
- Add prevention measures
